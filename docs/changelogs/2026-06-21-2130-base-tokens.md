# 2026-06-21 21:30 — Base Tokens (BT) normalized cost unit

## Task

Introduce a "Base Token" (BT) unit — `1 BT = the USD price of 1 input token of
claude-haiku-4-5` — and surface it across the dashboard: a "BT" column in the Live
Sessions table, BT as the Overview "Total tokens used" headline, and BT tooltips on
the Live Sessions summary cards (Input / Output / Cost). BT converts every token
type of every model into one comparable unit, like a base currency.

## Design / decisions

- **Formula:** `BT = costUsd / inputCostPerToken(claude-haiku-4-5)`, derived at read
  time. No DB schema change, no migration — a pricing refresh retroactively corrects
  historical BT.
- Base model is a hardcoded constant `BASE_TOKEN_MODEL = "claude-haiku-4-5"`.
- BT computed on the **backend**; the dashboard only formats/displays.
- Overview converts the **live-session** portion to BT; CodeBurn CSV tokens stay raw.
- Spec: `docs/superpowers/specs/2026-06-21-base-tokens-design.md`
- Plan: `docs/superpowers/plans/2026-06-21-base-tokens.md` (+ `.tasks.json`)

## What changed

**`packages/api` (PricingService — `pricing/pricing.service.ts`)**
- `BASE_TOKEN_MODEL`, `getBaseTokenRate()`, `toBaseTokens(costUsd, baseRate)`.
- `getModelRates(model)` + private `findPricing(model)` (extracted fuzzy lookup).
- `effectiveRates(p)` — single source of truth for the cache-read 10% /
  cache-creation 125% defaults; `calculateCost` refactored to reuse
  `findPricing` + `effectiveRates` (behavior unchanged, its 6 tests intact).
- Redis-cached pricing lookups (`findPricingCached`, key `pricing:model:<model>`,
  positive 1h / negative 5m TTL, cache-failure-safe → falls back to DB).

**`packages/api` (AgentSessionService — `agent-sessions/agent-sessions.service.ts`)**
- `list()`: per-row `baseTokens = toBaseTokens(costUsd, baseRate)` (base rate
  fetched once, folded into the `Promise.all`).
- `aggregate()`: combined `by:["model"]` groupBy (cost + token sums); returns
  `baseTokens: { input, output, total } | null`. `total` is anchored to
  `totalCostUsd / baseRate` (so it reconciles with the per-row column and Overview);
  the input/output split is the token-priced ratio scaled to that total.

**`packages/api` (StatsService — `stats/stats.service.ts`)**
- `getOverview()` returns `totalBaseTokens` (live-session cost ÷ base rate, null when
  unavailable); `totalTokensUsed` unchanged. `StatsModule` imports `PricingModule`.

**`packages/dashboard`**
- Types: `AgentSessionRow.baseTokens`, aggregate `baseTokens`,
  `OverviewStats.totalBaseTokens`.
- `formatBaseTokens(bt)` helper (rounds, `—` when null).
- "BT" column in `LiveSessionsTable`; BT tooltips on `SessionSummaryCards`
  (Input/Output/Cost); BT headline in the Overview "Total Tokens Used" card.

## Process

Built via subagent-driven development: 9 planned tasks + 1 review-fix task, each with
TDD, spec-compliance review, and code-quality review. A final holistic review caught
a cross-cutting issue (aggregate BT was recomputed from current pricing while the
per-row column used stored cost — they could disagree on one page); fixed by anchoring
the aggregate total to stored cost.
